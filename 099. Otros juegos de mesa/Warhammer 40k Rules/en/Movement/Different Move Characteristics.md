# Different Move Characteristics

## Summary

Different Move Characteristics explains how a unit resolves movement distance when not all of its models share the same Move Characteristic.

## Definition

When a unit makes a normal or advance move and its models have different Move Characteristics, each model uses its own maximum distance.

## Rules

Each model in the unit checks its own Move Characteristic when resolving how far it can move.

The unit still resolves the move as one movement event under the normal movement rules.

All other movement restrictions and end-state requirements still apply to the unit.

## Interactions

- [[Moving Units]] provides the baseline move procedure that this article refines for mixed-distance units.
- [[Move Characteristic]] is the glossary value each model uses when determining its own movement cap.
- [[Normal Move]] and [[Advance Move]] are the move types where this rule is explicitly applied.
- This article changes maximum distance calculations, but it does not replace coherency, engagement, terrain, or end-of-move legality checks.

## Restrictions

- Different Move Characteristics does not create a separate move type.
- This rule does not let one model's higher value override another model's lower value.
- This rule does not remove the requirement for the unit to finish the move legally.
