# Moving Vertically

## Summary

Moving Vertically is the terrain-movement procedure that explains how models ascend and descend terrain features and how that vertical travel contributes to total movement distance.

## Definition

Models can ascend or descend terrain features by moving vertically while remaining within 1/2" horizontally of that terrain feature.

## Rules

To move vertically, the model must remain within 1/2" horizontally of the relevant terrain feature while ascending or descending.

Add vertical distance moved upward and downward to the model's total distance moved.

This vertical travel is still part of the same underlying movement procedure.

Eligible units using [[Flying Models]] can ignore vertical distance if they take to the skies.

## Interactions

- [[Moving Units]] provides the baseline movement procedure that vertical movement extends rather than replaces.
- [[Ending a Move on Terrain]] determines whether the model can legally finish on the surface it reached.
- [[Flying Models]] is the primary override that changes how vertical distance is counted.
- [[Terrain Visibility System]] is the adjacent system map for terrain-related constraints and cross-links around this procedure.

## Restrictions

- A model cannot treat vertical movement as free distance unless a more specific rule allows it.
- The model must remain close enough to the terrain feature while ascending or descending.
- This article does not decide which terrain categories can be moved through; it only explains how vertical travel is measured.
