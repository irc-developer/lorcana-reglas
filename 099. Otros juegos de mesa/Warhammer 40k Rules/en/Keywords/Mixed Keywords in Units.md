# Mixed Keywords in Units

## Summary

Mixed Keywords in Units is the rule that determines how a unit uses the combined keywords of its component models while each model keeps only its own keywords.

## Definition

When a unit contains models with different keywords, the unit has all of those keywords, but each model keeps only its own keywords.

## Rules

If models in the same unit do not share the same keywords, the unit is treated as having the full set of keywords across all of its component models.

Each model still keeps only the keywords on that model and does not gain keywords from the other models in the unit.

This includes attached units and units whose datasheets or wargear create model-level keyword differences.
