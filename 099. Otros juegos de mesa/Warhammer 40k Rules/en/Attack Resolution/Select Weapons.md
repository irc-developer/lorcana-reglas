# Select Weapons

## Summary

Select Weapons is the opening attack procedure that determines which weapons each attacking model will use before any targets are chosen.

## Definition

Select Weapons is the step in which each model in the attacking unit chooses which weapons it will use for the current attack.

## Rules

Select Weapons happens inside [[Making Attacks]] before targets are declared.

While shooting, a model can select one or more ranged weapons.

While fighting, a model selects one melee weapon unless another rule changes that default.

If a model has no ranged weapons, it cannot make ranged attacks.

If a model has no melee weapons, it cannot make melee attacks.

The chosen weapons determine which attacks proceed into [[Select Targets]] and later into [[Resolve Attacks]].

## Interactions

- [[Making Attacks]] is the parent procedure that requires weapon selection before target selection.
- [[Select Targets]] uses the chosen weapons to determine legal enemy targets.
- [[Weapon System]] is the published upstream system for weapon profiles, weapon characteristics, and weapon abilities consumed here.
- [[Shooting System]] and [[Fight System]] are the phase-level surfaces that feed models into this shared weapon-selection step.

## Restrictions

- A model cannot select a weapon it does not have.
- This procedure does not itself create permission to attack; it only determines which available weapons are being used.
- Any rule that changes the normal number or type of weapons a model can use must say so explicitly.
