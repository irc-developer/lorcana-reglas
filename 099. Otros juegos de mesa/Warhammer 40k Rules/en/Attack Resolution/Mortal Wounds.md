# Mortal Wounds

## Summary

Mortal Wounds is the direct wound-loss rule used when an effect inflicts wounds outside the normal save sequence.

## Definition

Mortal wounds are wounds inflicted directly on a unit outside the normal save sequence.

## Rules

When a unit suffers mortal wounds, those wounds are allocated to a model using the stated priority.

Allocate first to a wounded non-CHARACTER model if one is available.

If no such model is available, allocate to another non-CHARACTER model.

If no non-CHARACTER model is available, allocate to a wounded CHARACTER model.

If none of those apply, allocate to another CHARACTER model.

Each mortal wound causes the selected model to lose 1 wound.

When an attack inflicts both normal damage and mortal wounds, resolve the normal damage first and the mortal wounds afterwards.

## Interactions

- [[Inflict Damage]] is the normal damage branch that mortal wounds bypass or follow, depending on the rule causing them.
- [[Hazard Rolls]] is a published procedure that causes mortal wounds on a failed roll.
- [[Destroyed]] becomes relevant when accumulated mortal wounds reduce a model to 0 wounds.
- [[Attack System]] places mortal wounds adjacent to the main attack pipeline because attack abilities and some procedures can feed into them.
- [[Attached Unit System]] remains relevant when direct wound allocation interacts with unit structure and model priority.

## Restrictions

- Mortal wounds are not resolved through the normal save sequence.
- The allocation priority must be followed instead of assigning mortal wounds freely.
- When a rule combines normal damage and mortal wounds from the same attack, the mortal wounds do not resolve first.
