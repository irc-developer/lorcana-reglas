# Save Rolls

## Summary

Save Rolls is the defensive step in the attack sequence where the opposing player groups the target unit, fixes allocation order, and rolls saves against successful wounds.

## Definition

The opposing player groups the target unit, declares allocation order, and rolls one D6 for each successful wound.

## Rules

Save Rolls happens after [[Wound Rolls]] and before [[Inflict Damage]].

Create one group for each CHARACTER model.

Create one group for all other models that share the same Wounds characteristic, Save characteristic, and Invulnerable Save characteristic.

Non-CHARACTER groups come before CHARACTER groups.

Where the rule requires it, wounded groups are placed earlier in the allocation order.

Once the groups and order are fixed, roll one D6 for each successful wound.

The resulting save values are then processed in [[Inflict Damage]].

## Interactions

- [[Wound Rolls]] supplies the successful wounds that now require saving throws.
- [[Inflict Damage]] uses the save results and the declared allocation order to determine whether models lose wounds.
- [[Attacking Attached Units]] and [[Attached Unit System]] matter when the target unit's structure affects how attacks are resolved against it.
- [[Attack Sequence]] places save rolls between offensive wound generation and actual damage application.

## Restrictions

- Save grouping and allocation order are declared before damage is applied.
- CHARACTER models do not merge into the shared non-CHARACTER groups.
- This step does not itself apply damage; it prepares and rolls the defensive results that [[Inflict Damage]] resolves.
