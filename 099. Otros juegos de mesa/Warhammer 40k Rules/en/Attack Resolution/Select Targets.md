# Select Targets

## Summary

Select Targets is the attack-declaration procedure that assigns each selected weapon to the enemy unit or units it will attack.

## Definition

For each selected weapon, choose the target unit or units that are allowed for the current attack type.

## Rules

Select Targets happens after [[Select Weapons]] and before [[Resolve Attacks]].

While shooting, targets must normally be visible, within range, and unengaged unless another rule changes that default.

While fighting, each target must be engaged with the attacking model.

When a melee weapon is split across more than one target, that split must be declared at this step.

The total number of melee targets chosen for a weapon cannot exceed that weapon's Attacks characteristic.

Once targets are fixed, the declared attacks proceed into [[Resolve Attacks]].

## Interactions

- [[Select Weapons]] provides the selected weapons that now need legal targets.
- [[Resolve Attacks]] uses the declared target assignments and resolves them one enemy unit at a time.
- [[Terrain Visibility System]] supplies the published visibility surface used when shooting target legality depends on sight and cover-related states.
- [[Shooting System]] and [[Fight System]] supply the phase-specific permission context for this shared targeting step.

## Restrictions

- This procedure does not ignore visibility, range, [[Engagement|engagement]], or any other targeting restriction unless a rule says otherwise.
- Target splits must be declared before the attacks are resolved.
- Target legality is determined per selected weapon, not only once for the unit as a whole.
