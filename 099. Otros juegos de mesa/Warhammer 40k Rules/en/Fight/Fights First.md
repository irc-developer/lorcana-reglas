# Fights First

## Summary

Fights First is the fight-order state that moves a unit into the earlier melee sequencing window before remaining combats are resolved.

## Definition

A unit is a Fights First unit while every model in it has this ability.

## Rules

This concept changes the order in which eligible units are selected during melee resolution.

Fights First combats are resolved before the remaining combats.

If new Fights First units become eligible while remaining combats are being resolved, melee sequencing returns to the Fights First window before continuing.

Charge-derived permissions and effects such as Counteroffensive can create this state, but they do not transfer ownership of the state out of Fight.

## Interactions

- [[Fight]] consumes this state to order which eligible units are selected first.
- [[Fights First unit]] is the glossary support surface for the unit state created when every model has this ability.
- [[Charge System]] is an upstream neighbouring system because charge move outcomes can grant this state until end of turn.

## Restrictions

- This article defines sequencing state, not charge declaration, charge movement, or attack resolution.
- This article does not replace the unit-selection logic in [[Fight]]; it modifies that logic by changing priority.
- Unpublished ability-granting and stratagem-granting surfaces remain outside this page and should be referenced from their own domains when they are published.
