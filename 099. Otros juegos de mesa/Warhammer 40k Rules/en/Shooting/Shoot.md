# Shoot

## Summary

Shoot is the Shooting-domain procedure where a unit is selected to resolve one eligible type of ranged attack during the Shooting phase.

## Definition

To shoot is to select a unit in the Shooting phase and resolve its attacks using one eligible shooting type.

## Rules

During the Shooting phase, units shoot one at a time.

When a unit shoots, it uses one eligible shooting type for that selection.

The detailed attack procedure is resolved through [[Making Attacks]] rather than being redefined here.

After a unit is selected to shoot, that unit is no longer eligible to start an action until the end of the phase.

## Interactions

- [[Shooting Phase]] is the canonical Game Sequence timing article that contains this procedure.
- [[Battle Sequence System]] provides the wider timing spine above that phase-level owner.
- [[Shooting System]] maps the phase-specific shooting permissions and restrictions that determine which shooting type is available.
- [[Attack System]] and [[Making Attacks]] provide the shared attack pipeline consumed once a unit is allowed to shoot.
- [[Normal Shooting]], [[Assault Shooting]], [[Close-quarters Shooting]], and [[Indirect Shooting]] are the published shooting modes that branch from this procedure.
- [[Snap Shooting]] is the published special shooting procedure used only when another rule grants that instruction.

## Restrictions

- This procedure does not redefine the attack sequence, target selection rules, or weapon data used by the attacks it permits.
- A unit must have an eligible shooting type before it can be selected to shoot.
- This procedure only covers the act of selecting and resolving a unit's shooting, not the entire phase timeline.
