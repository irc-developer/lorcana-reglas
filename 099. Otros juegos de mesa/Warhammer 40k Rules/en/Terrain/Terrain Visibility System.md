# Terrain Visibility System

## Overview

Terrain Visibility System is the authoritative System Article for the Terrain domain. It organises the published terrain concepts, glossary support, and boundary rules that define battlefield features, terrain areas, visibility state, cover, and terrain-origin restrictions.

## Scope

This domain covers terrain placement, terrain categories, terrain areas, the default and terrain-modified visibility model, cover, obscuring, hidden state, solid terrain, and the glossary support those concepts require.

This domain helps answer questions about what terrain is, how terrain changes the battlefield state, and which other systems consume that terrain state.

## Domain Ownership

- Owner domain: Terrain.
- Domain role: publish and maintain the canonical terrain concepts that describe battlefield features, terrain areas, visibility state, cover, and terrain-origin restrictions.
- Publication boundary: this page is a navigation hub. It does not replace the canonical child articles that define the individual terrain rules.

## Domain Boundaries

- Terrain owns terrain features, terrain areas, visibility, cover, terrain categories, terrain restrictions, and terrain-based interactions.
- Terrain does not own movement procedures such as [[Normal Move]], [[Advance Move]], or [[Fall-back Move]].
- Terrain does not own the attack pipeline such as [[Attack Sequence]], [[Hit Rolls]], [[Wound Rolls]], or [[Save Rolls]].
- [[Movement System]] consumes terrain restrictions and terrain traversal outcomes, but Movement remains owner of move procedures.
- [[Attack System]] and [[Shooting System]] consume visibility and cover state, but Terrain remains owner of that battlefield state.
- [[Objective Control System]] consumes terrain areas when objectives are represented through terrain rather than markers.

## Navigation

- Start with [[Visibility]] when the question is about the default line-of-sight rule before terrain modifies it.
- Start with [[Placing Terrain]] and [[Terrain area]] when the issue is how terrain enters the battlefield state.
- Read [[Terrain Categories]], [[Exposed]], and [[Light]] when the issue is how a terrain feature is classified.
- Read [[Terrain and Visibility]] when the question is about the terrain-owned visibility umbrella rather than a single child rule.
- Continue to [[Benefit of Cover]], [[Hidden]], [[Obscuring]], or [[Solid]] for the specific terrain-derived battlefield state or restriction you need.
- Read [[Visible]], [[Fully visible]], [[Obscuring terrain area]], or [[Detection range]] when the question is a glossary distinction.
- Move to [[Movement System]] when the real issue is how a move is performed.
- Move to [[Attack System]] or [[Shooting System]] when the real issue is how an attack consumes the resulting terrain state.
- Move to [[Objective Control System]] when the question is about how terrain areas function as objectives.
