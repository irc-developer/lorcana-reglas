# Visibility

## Summary

Visibility is the baseline battlefield rule that determines whether an observing model has line of sight to another model before terrain-specific modifiers are applied.

## Definition

An observing model has line of sight if a straight 1 mm line can be drawn from any part of that model to any part of the observed model, ignoring models in the observing unit and observed unit.

## Rules

A model is visible if any part of it is visible.

A model is fully visible if every part facing the observer is visible, with only itself blocking itself.

A unit is visible if one or more models in it are visible.

A unit is fully visible if every model in it is fully visible.

Terrain can change visibility through [[Hidden]], [[Obscuring]], and [[Solid]].

## Interactions

- [[Attack System]] consumes visibility when attacks require an eligible visible target, but it does not redefine line of sight.
- [[Terrain and Visibility]] is the terrain-owned branch that explains how battlefield features modify the default visibility rule.
- [[Visible]] and [[Fully visible]] provide the glossary support used by cover and target-eligibility checks.

## Restrictions

- Visibility defines battlefield state; it does not grant a permission to shoot, fight, or move.
- Terrain-derived exceptions and blockers belong to the Terrain domain and are applied through the terrain articles, not by changing the core line-of-sight test here.
- Other non-terrain exceptions to visibility remain external rules and are not redefined on this page.
