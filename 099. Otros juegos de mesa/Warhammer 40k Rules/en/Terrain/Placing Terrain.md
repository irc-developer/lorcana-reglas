# Placing Terrain

## Summary

Placing Terrain defines how terrain features are represented on the battlefield and how they create the terrain areas that later rules use.

## Definition

Terrain features are placed by boundary, directly on the battlefield, or by forming a terrain area with multiple features.

## Rules

The area occupied by that placement is the [[Terrain area]].

Mission deployment maps may define the location and size of terrain areas.

If a mission does not define them, players agree terrain areas before the battle.

This placement step establishes the battlefield state that later terrain, objective, and visibility rules consume.

## Interactions

- [[Terrain area]] names the occupied battlefield space created by terrain placement.
- [[Terrain Categories]] classifies the terrain features that have been placed.
- [[Objective Control System]] later consumes terrain areas when objectives are represented through terrain instead of markers.

## Restrictions

- This article defines terrain placement state, not movement procedures through terrain.
- This article does not define attack permissions, cover resolution, or objective-control calculations.
- Mission-specific deployment instructions can constrain where terrain is placed, but those instructions remain outside the Terrain domain.
