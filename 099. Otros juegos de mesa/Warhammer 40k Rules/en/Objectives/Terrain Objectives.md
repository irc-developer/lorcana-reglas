# Terrain Objectives

## Summary

Terrain Objectives is the default objective-representation rule that treats an objective as a terrain area at the mission's stated location point.

## Definition

If a mission uses objectives, the objective is normally the terrain area at the stated location point.

## Rules

When an objective is represented through terrain, the objective is the terrain area itself rather than a separate marker.

Distances to and from that objective are measured to and from the closest part of the terrain area.

If the objective's location point does not coincide with a terrain area, use [[Objectives Not Within a Terrain Area]] instead.

## Interactions

- [[Terrain area]] is the terrain-owned battlefield state consumed when an objective is represented through terrain.
- [[Level of Control]] uses this article's representation rule to determine when a model is within range of a terrain objective.
- [[Objectives Not Within a Terrain Area]] replaces this representation branch when the mission location point does not coincide with a terrain area.
- [[Terrain Visibility System]] remains owner of terrain areas and other terrain state even when those areas function as objectives.

## Restrictions

- This article does not define terrain placement, terrain ownership, visibility, or cover.
- It only defines the default objective representation when a mission uses objectives and the stated location point coincides with a terrain area.
- It does not replace the separate marker-based branch defined in [[Objectives Not Within a Terrain Area]].
