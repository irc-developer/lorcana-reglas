# Units and Models

## Summary

Units and Models is the core-concepts rule that defines a unit as one or more models acting together as a single group and establishes that rules applied to a unit apply to every model in it.

## Definition

A unit contains one or more models that move and fight together as a single group.

A model's base is part of that model for rules purposes unless a rule states otherwise.

## Rules

Friendly units and friendly models are units and models in your army.

Enemy units and enemy models are units and models in your opponent's army.

If a rule affects units or models without specifying friendly or enemy, it affects any valid unit or model.

A rule that applies to a unit applies to every model in that unit.

## Interactions

- Army provides the ownership context that distinguishes friendly units and models from enemy units and models.
- [[Mixed Keywords in Units]] depends on this distinction because the unit can have a wider rules state than any individual model within it.
- [[Attached unit]] is a downstream glossary concept for a single combined unit, but the unit/model distinction defined here still determines what counts as the unit and what counts as its component models.
- [[Forming Attached Units]] is a specialized cross-domain procedure that creates one unit from multiple model-bearing components without changing this article's general unit/model rule.

## Restrictions

- This article defines only the general distinction between a unit and a model and does not restate keyword aggregation, attached-unit formation, movement procedures, battle-shock effects, or abilities.
- A rule written for a model is not automatically a rule written for the whole unit unless the source rule says so.
- A model's base counts as part of that model only unless a rule states otherwise.
