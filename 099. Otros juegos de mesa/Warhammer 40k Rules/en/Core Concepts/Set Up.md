# Set Up

## Summary

Set Up is the battlefield-placement procedure that places a unit on the battlefield while preserving coherency, unengaged state, and every other applicable restriction.

## Definition

When setting up a unit, place its models on the battlefield so that the unit is in [[Coherency]], unengaged, and obeys all other restrictions.

## Rules

The unit must be placed so that it is in [[Coherency]] when setup finishes.

The unit must be unengaged when setup finishes.

All other restrictions that apply to the setup must also be obeyed.

If you cannot set up all models in the unit, return that unit to its original position.

If this happens during a move type, the unit is treated as not having been selected for that move.

## Interactions

- [[Coherency]] defines the formation requirement that setup must satisfy.
- [[Engagement]] defines the engaged and unengaged state that setup must respect.
- [[Revived]] consumes this procedure when returned models are placed back into their unit.
- [[Ingress Move]] is a reserve-arrival procedure that uses setup rather than the generic in-battle movement sequence.
- [[Movement System]] consumes this article as the baseline placement rule that several move-adjacent procedures rely on.
- [[Strategic Reserves System]] consumes this article for reserve arrival without taking ownership of the setup rule itself.

## Restrictions

- A unit cannot be successfully set up unless every model in that unit can be placed legally.
- If legal placement fails, the unit returns to its original position.
- If the failed setup happened during a move type, the unit is treated as not having been selected for that move.
- This procedure does not redefine reserve permissions, embark/disembark rules, or generic movement pathing.
