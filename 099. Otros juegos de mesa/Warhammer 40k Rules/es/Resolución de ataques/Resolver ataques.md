# Resolver ataques

## Resumen

Resolver ataques es el paso que convierte las armas y blancos declarados en dados de ataque contra unidades enemigas concretas.

## Definición

Para resolver ataques, el jugador activo elige una unidad enemiga que haya sido blanco, reúne los dados de ataque de armas que atacan a esa unidad y resuelve la [[Secuencia de ataque]] para esos dados.

## Reglas

El procedimiento se resuelve así:

1. Elegir una unidad enemiga que haya sido blanco de una o más armas.
2. Elegir un arma que haya tomado esa unidad como blanco y que aún no haya atacado a esa unidad.
3. Reunir una cantidad de D6 igual al atributo A de esa arma. Esos dados son dados de ataque.
4. Añadir, si corresponde, los dados de otras armas que hagan ataques idénticos contra esa misma unidad y que aún no hayan atacado a ese blanco.
5. Resolver la [[Secuencia de ataque]] para todos los dados reunidos.
6. Repetir el proceso para otras armas contra la misma unidad o para armas con ataques pendientes contra otra unidad.

## Ataques idénticos

Los ataques idénticos son aquellos que tienen los mismos atributos HP/HA, F, FP y D, y a los que afectan las mismas habilidades y reglas aplicables.

## Interacciones

- [[Elegir blancos]] determina qué unidades enemigas pueden ser elegidas aquí.
- [[Secuencia de ataque]] procesa los dados reunidos.
- Las armas que dividen ataques de combate solo reúnen los dados declarados contra la unidad blanco actual.

## Restricciones

- No puedes resolver ataques de un arma contra una unidad que no fue declarada como blanco de esa arma.
- Los ataques sobrantes no se reasignan automáticamente a otro blanco.
- Este paso termina cuando todas las armas han realizado todos sus ataques.

## Navegación

- Usa [[Secuencia de ataque]] para resolver los dados reunidos.
- Vuelve a [[Elegir blancos]] si la duda es si una unidad podía ser blanco.
- Vuelve a [[Elegir armas]] si la duda es qué armas estaban disponibles.
